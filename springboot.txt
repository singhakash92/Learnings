4) Dependency Injection:
class controller ( internally "depends" on a function inside service)[so, in-order to use the function of service it needs a object of service first, we will injection this Dependency in controller(no need to create the obj, we are taking away the power of creating object in controller here)]
class service
class repository

a)Constructor Injection: the moment we create the object of controller we also pass the object of service as a argument, we receive it and set the field for service, now we can use the function of service class // use to inject mandatory Dependency
b)setter Injection: we create obj of controller, now we have a setter function inside controller to set the field of service, we class that function and pass the service object as an argument //optional Dependency //write @Autowired(required = false) above the setter
c)field Injection: simply write @autowire above the object declared where ever we want to user the function of service //never use in prod

6)ObjectMapper class in Jackson library(********Used for JSON mapping with java Object vise versa)#####DTO creation
used for Serializing -> java object to JSON/xml
        deserializing -> json to java object

public class Employee {

    @JsonProperty("emp_id")        //user sending emp_id in i/p request, now it's mapped to id for processing, while retring it's again mapped to emp_id  
    private int id;                  // while working internally we use id

    @JsonIgnore                       // exclude from serialization and deserializing
    private String password;          // when json is send back to user this filed is not present there

    @JsonAlias({"emp_name", "name"})  // accept multiple JSON key names coming from user during deserializing but returned is name
    private String name;

    @JsonFormat(pattern = "yyyy-MM-dd")
    private LocalDate joiningDate;

    @JsonInclude(JsonInclude.Include.NON_NULL) //Exclude null or empty fields while returning the json
    private String email;

    @JsonIgnoreProperties(ignoreUnknown = true)   //class level, json coming from user, not here so ignore
    public class Employee {
    }

    Annotation	Purpose
    @JsonProperty	Rename JSON field coming from user and while sending back
    @JsonIgnore	Ignore field coming from user
    @JsonAlias	Accept alternate names coming from user
    @JsonFormat	Format dates/numbers while sending to user
    @JsonInclude	Exclude null/empty while sending to user
    @JsonIgnoreProperties	Ignore unknown fields coming from user
    JsonGetter
    JsonSetter ///very good to learn
}

##############about entities
1)Entity ->	java object that represent a database table row
2)Entity Lifecycle -> states of entity, moves through 4 states
    transient - enitity created, but not geting tracked,   User user = new User();
    persistent - enitity getting tracked, entityManager.find(User.class, 1);Stored in Persistence Context and Dirty checking ON
    detached - no longer is getting tracked, entityManager.detach(user);
    removed - marked for deletion , entityManager.remove(user);
3)EntityManager	: this is what we use to interact with DB
e.g. 
entityManager.persist();
entityManager.find();
entityManager.remove();
entityManager.createQuery();


core of JPA
a)persistent context - tracks managed Entity
    Stores managed entities
    Prevents duplicate DB calls
    Tracks changes
b)dirty checking - auto detect changes and update DB
c)query generation - convert jPQL to sql
d)caching - avoid unnecessary db hit by caching
